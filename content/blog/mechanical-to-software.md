# 從機械工程到軟體開發：跨域轉職心路歷程

## 起點：汽車修護科

我的學習起點是汽車修護，每天與引擎、變速箱為伍。那時候，「程式設計」對我來說是另一個世界。

## 轉折點

2021 年，一次偶然的機會接觸到 Arduino，開始了自學程式之旅。發現原來程式可以讓機械「活」起來。

## 學習路徑

```
2021 Q1: Arduino + C 基礎
2021 Q3: Python 入門
2022 Q1: Web 開發 (HTML/CSS/JS)
2022 Q3: Vue.js + Flask
2023 Q2: AI 應用開發
2024 Q1: 雲端部署與 DevOps
```

## 關鍵轉變

### 思維模式

機械工程講究「精準」，軟體開發講究「迭代」。學會接受「先做出來再優化」的開發哲學。

### 學習方法

- **專案驅動**：有明確目標才有動力
- **社群參與**：GitHub、技術論壇積極互動
- **持續輸出**：寫部落格整理所學

## 跨域優勢

1. **系統思維**：機械背景讓我擅長拆解複雜問題
2. **耐心調試**：修車經驗讓我不怕 Debug
3. **硬體整合**：IoT 開發得心應手

## 給跨域者的建議

> 不要怕起步晚，重要的是持續前進。

每個領域都有可轉移的技能，找到連結點，跨域就不是障礙而是優勢。
